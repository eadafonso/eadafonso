  <h2 className="font-semibold">1- O Verdadeiro Papel do Front-End: Muito Além das Telas Bonitas</h2>

{" "}

<span className="tracking-normal text-slate-700">
  Quando pensamos em desenvolvimento front-end, é comum imaginar que a principal
  responsabilidade é criar interfaces visuais, elementos bonitos e atrativos
  para os utilizadores. Contudo, o front-end vai muito além de simples telas
  estéticas; o seu verdadeiro propósito é criar experiências de utilizador
  envolventes, intuitivas e acessíveis. O trabalho de um front-end é garantir
  que a interação do utilizador com a aplicação seja agradável, eficiente e
  adaptada para todos os públicos, incluindo aqueles com necessidades especiais.
</span>

{" "}

<h3 className="font-medium">
  2- Front-End: Mais do que Apresentação, uma Experiência Completa
</h3>
<span className="tracking-normal text-slate-700">
  O desenvolvimento front-end envolve pensar em acessibilidade, usabilidade e
  experiência de utilizador (UX). Trata-se de desenhar aplicações que facilitem
  a navegação e ofereçam uma experiência sem frustrações ou barreiras. Cada
  clique, cada scroll e cada mudança de página são oportunidades para melhorar
  ou piorar a experiência de quem está a usar o site.
</span>

{" "}

<h3 className="font-medium">2.1- Acessibilidade: Uma Experiência para Todos</h3>
<span className="tracking-normal text-slate-700">A acessibilidade é um dos pilares fundamentais de uma boa aplicação. 
Um site acessível permite que pessoas com deficiências visuais, motoras ou cognitivas possam navegar e interagir sem dificuldades. 
Isso significa pensar na utilização de cores contrastantes, alternativas de texto para imagens, navegação por teclado e estruturação semântica dos elementos. 
Estes detalhes não apenas beneficiam pessoas com necessidades especiais, mas também melhoram a experiência para todos, pois um site acessível é muitas vezes mais intuitivo e claro para qualquer utilizador.</span>

  <h3 className="font-medium">2.2- Experiência de Utilizador: Cada Detalhe Conta</h3>
  
  <span className="tracking-normal text-slate-700">
  Imagina a situação: entras num site de comércio electrónico (E-commerce) para procurar um produto específico. Navegas por várias secções, fazes scroll e finalmente encontras o item que procuras. 
  Abres a página do produto, verificas os detalhes e, quando decides voltar à página anterior para explorar outras opções, és redirecionado para o topo da página. 
  Terás de fazer todo o scroll novamente, procurando onde estavas anteriormente. Esse tipo de frustração pode parecer pequena, mas é suficiente para afastar um utilizador e até levá-lo a desistir de usar o site. <br /> <br />

  Para evitar isso, o front-end deve focar-se em detalhes que muitas vezes passam despercebidos, mas que fazem uma diferença significativa na experiência do utilizador. Guardar o estado de scroll, por exemplo, é uma prática simples mas poderosa para tornar a navegação mais agradável.
  Pequenos detalhes como este mostram respeito pelo tempo e paciência do utilizador.

  </span>

{" "}
<h3 className="font-medium">3- Ferramentas e Boas Práticas para um Front-End de Qualidade</h3>

<span className="tracking-normal text-slate-700">
Para criar uma experiência completa, é essencial usar ferramentas e adotar boas práticas que ajudem a construir interfaces intuitivas e acessíveis:
</span>
<div className="flex flex-col gap-3">
  <span>
    <span className="font-medium">3.1- Componentes Reutilizáveis:</span>{" "}
    Criar componentes que possam ser reutilizados em várias secções melhora a consistência da interface e facilita a manutenção do código.
  </span>

 <span>
    <span className="font-medium">3.2- Feedback Visual:</span>{" "}
  Sempre que um utilizador interage com a página, como ao clicar num botão ou enviar um formulário, é importante dar um feedback visual imediato. 
  Isso confirma que a ação foi registada e evita frustrações.
  </span>

   <span>
    <span className="font-medium">3.3- Gestão de Estado:</span>{" "}
    Ferramentas como o React Query ou mesmo o contexto de estado no React ajudam a preservar o estado das páginas e componentes. 
    No caso de navegação entre páginas, o estado da posição do scroll ou da lista de produtos visualizados pode ser guardado para proporcionar uma experiência mais fluida.
  </span>

   <span className="pb-12">
    <span className="font-medium">4- Testes de Acessibilidade:</span>{" "}
   Usar ferramentas como o ANDI, Wave ou Lighthouse para identificar problemas de acessibilidade no site é essencial para garantir que a aplicação é acessível a todos.
  </span>
